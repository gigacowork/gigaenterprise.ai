Представляем первый релиз GigaCowork — платформы ИИ-агентов для всей компании. Это ранняя версия продукта, в которую мы заложили самое основное для решения повседневных и сложных бизнес-задач: от анализа документов до автоматизации повторяющихся сценариев.

Универсальный агент

Это агент с универсальными знаниями, с которым можно работать в любых пространствах. Он может выполнять разные задачи, для этого:

  • дополнительно активирует навыки, доступные пользователю
  • использует документы из сессии или пространства как дополнительный контекст
  • исполняет команду по инструкции

В каждой сессии формируется свой изолированный контекст для работы универсального агента.

Задачи

Агент начинает работу при постановке задачи. При создании каждой задачи запускается изолированная сессия.

В задачах предусмотрена работа с документами:

  • можно прикреплять файлы и папки к сообщению, чтобы система анализировала их в рамках текущей задачи;
  • все созданные ИИ-документы (таблицы, отчеты, тексты) сохраняются в панели компонентов справа и доступны для скачивания.

Ход выполнения задачи отображается в панели отслеживания прогресса прямо внутри задачи.

Вся история работы с задачами сохраняется и доступна в полном виде. Можно в любой момент просмотреть задачу, найти нужную информацию. И даже если задача решена, в сессию по ней можно вернуться и что-то доделать.

Навыки

Навыки — описанные обычными словами инструкции, по которым агент выполняет конкретные задачи. По навыку агент определяет способ выполнения задачи: например, как оформлять документы в фирменном стиле, каким образом анализировать продажи или предложения поставщиков.

Создавать и изменять навыки можно двумя способами: написать инструкцию или загрузить готовый файл с описанием.

Применение навыков агентами

  • Агент сам подбирает нужный из доступных в сессии навык по описанию, но при необходимости вы можете явно указать его в запросе задачи.
  • Если в навыке необходимо использование коннектора для интеграции с какой-либо системой или сервисом — достаточно указать название коннектора в описании инструкции.
  • При необходимости в навыке можно выполнять Python-скрипты, но есть ограничения по безопасности.

Команды (/-команды)

Команда — это сохраненный запрос с инструкцией. Если вы часто делаете одинаковые запросы к агентам, удобнее сохранить его в команду и вызывать при необходимости. В команде можно прописать подключение коннектора, запрос данных, расчет метрик и формат вывода — любые повторяющиеся действия.

Вызвать команду очень просто — достаточно ввести / в поле ввода или нажать кнопку «Вызвать команду или навык». Появится весь список команд, нужно выбрать подходящую.

Пространства

Пространства — это изолированные рабочие среды для отделов и команд с собственными агентами, документами и правилами доступа. Внутри пространства сохраняется весь контекст: документы, навыки, задачи и сессии.

Пространство может быть

Личное

Пользователь создает его сам для себя. Документы и настройки видны только ему.

Командное

Пространство для совместной работы, в которое можно добавлять участников. Документы и навыки становятся доступны всей команде.

Пространство можно настраивать и адаптировать под конкретные сценарии использования, добавляя контекст в виде документов, новых агентов или навыков и коннекторов.

Коннекторы для внешних систем

Коннекторы — безопасные подключения к внешним и корпоративным системам (CRM, ERP, почта, файловые хранилища, папки и документы) по протоколу Model Context Protocol (MCP). Он позволяет быстро расширять набор поддерживаемых систем без доработок действующей ИТ-инфраструктуры. Коннекторы настраиваются администратором один раз, а пользователи сами подключают необходимые для выполнения задачи, нажимая кнопку в окне чата.
